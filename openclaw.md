# openclaw.md

## Project brief
Build a MonoGame version of the JavaScript broughlike tutorial with a more extensible architecture.

## Delivery goals
1. Preserve the finished game's mechanics.
2. Keep code easy to extend with new monsters, spells, tiles, effects, and UI.
3. Maintain a documented path to a web build that can be hosted on GitHub Pages.

## Working rules
- Keep tutorial reference material under `docs/tutorial/`.
- Keep architecture notes under `docs/architecture/`.
- Prefer small, composable gameplay systems over tightly coupled tutorial-style globals.
- Validate changes with the smallest meaningful local run/build step.
- Update task/context files when milestones complete.
- Record GitHub delivery evidence and remaining web-publish blockers in `docs/architecture/`.
